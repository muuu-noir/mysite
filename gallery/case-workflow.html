Case 1 — EC経理計算の月次自動化

使用技術
Excel VBA
背景
エンタメ会社の国内外向けECサイトでは、月次の経理計算が複数シートにまたがる手作業で行われており、担当者の工数と入力ミスのリスクが課題でした。
対応
売上データの集計・通貨換算・帳票出力までの一連のフローをExcel VBAでマクロ化。月次のルーティン処理をボタン一つで完結できる仕組みに置き換えました。
成果
毎月発生していた手作業の繰り返しを排除。担当者が本来の業務に集中できる体制を整えました。

Case 2 — ExcelデータのAccess / Notion移行提案

使用技術
Excel / Access / Notion / プロセス設計
背景
書籍流通業界においてデータがExcelファイルで管理されており、検索・更新・共有のたびに手間がかかっていました。複数人が同じファイルを触ることによる上書きリスクも問題でした。
対応
データ構造を分析し、利用頻度・共有範囲・操作スキルに応じてAccessとNotionの2案を提案。業務フロー全体を再設計した上で移行を主導しました。
成果
3日かかっていた作業を半日に短縮。データの重複・上書きリスクもゼロに。

Case 3 — 30,000件データクリーニングの自動化

使用技術
Python
背景
友人からの依頼で、30,000件規模のデータクリーニングが必要な状況に。手作業で対応した場合、60〜100時間の工数が見込まれていました。
対応
Pythonでデータ検証・クレンジング・整形の処理を自動化するスクリプトを設計・実装。対象データの構造を分析し、エラーパターンを洗い出した上でロジックを組みました。
成果
手作業60〜100時間の処理を約5時間で完了。品質も手作業より均一に保てました。
